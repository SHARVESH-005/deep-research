# Deep Research Autonomous Multi-Agent Synthesizer

A "Perplexity-style" or "Gemini Deep Research" engine that orchestrates multiple AI agents to perform complex, multi-step web research. 

This project uses **LangGraph** to route queries through a Planner, Searcher, and Synthesizer node in a pipeline. The backend streams real-time state updates to a **Next.js** frontend via **WebSockets**, generating a final, beautifully formatted research report with inline citations and a confidence score.

## Features
- **Multi-Agent Architecture**: Uses `LangGraph` for orchestrating specialized agents (Planner, Searcher, Synthesizer).
- **Live Web Research**: Uses the `Tavily API` to browse the web for up-to-date and accurate context.
- **Real-Time Streaming**: Uses `FastAPI` WebSockets to stream agent state to the frontend in real-time.
- **Zero Hallucination Grounding**: The Synthesizer is forced to cite the retrieved context, preventing hallucinated answers.
- **Clean UI**: A minimalistic, highly responsive Next.js and TailwindCSS interface.

## Tech Stack
- **Backend**: Python, FastAPI, LangGraph, LangChain, Google Gemini API (gemini-2.5-flash), Tavily Search API.
- **Frontend**: Next.js 16 (App Router), React, Tailwind CSS, React Markdown.
- **Infrastructure**: Docker Compose (ready for Redis caching).

## 🚀 Getting Started

### Prerequisites
1. Python 3.10+
2. Node.js 18+
3. API Keys for [Google Gemini](https://aistudio.google.com/) and [Tavily](https://tavily.com/).

### 1. Backend Setup

Navigate to the `backend` directory (located in the root of the repository):
```bash
cd ../backend
```

Create a virtual environment and install dependencies:
```bash
python -m venv venv
.\venv\Scripts\activate  # On Windows
# source venv/bin/activate  # On macOS/Linux
pip install -r requirements.txt
```

Set up your `.env` file:
Create a `.env` file in the `backend` folder and add your API keys:
```env
GOOGLE_API_KEY=your_gemini_api_key_here
TAVILY_API_KEY=your_tavily_api_key_here
REDIS_URL=redis://localhost:6379
```

Start the FastAPI server:
```bash
python main.py
```
*The backend will run on `http://localhost:8000` and listen for WebSockets at `ws://localhost:8000/ws/research`.*

### 2. Frontend Setup

Open a new terminal and navigate to the `frontend` directory:
```bash
cd frontend
```

Install the dependencies:
```bash
npm install
```

Start the Next.js development server:
```bash
npm run dev
```
*The frontend will run on `http://localhost:3000`.*

### 3. Usage
1. Open your browser to `http://localhost:3000`.
2. Type a complex research query (e.g., "What are the latest updates on SpaceX Starship?").
3. Watch as the UI streams the progress of the Planner, Searcher, and Synthesizer agents.
4. Read the final cited report generated by the agents.

## Architecture details

1. **Planner Node**: Receives the user's query and uses Gemini to break it down into 2-3 specific, actionable search queries.
2. **Searcher Node**: Takes the sub-queries and uses the Tavily API to retrieve the most relevant web pages, deduplicating the sources.
3. **Synthesizer Node**: Feeds all the scraped context into Gemini with a strict system prompt to write a structured report, generate inline citations, and assign a confidence score.

## Future Improvements
- **True Parallel Searching**: Execute Tavily searches asynchronously to reduce overall latency.
- **Token Streaming**: Stream the generated report text block-by-block instead of waiting for the full response.
- **Redis Caching**: Uncomment the Redis cache lines in `agent.py` and spin up the `docker-compose.yml` stack to cache redundant Tavily searches.
